# Polarity-Aware LLM Query Expansion for Financial Opinion Search

## Overview

This project investigates how sentiment-controlled query expansion generated by Large Language Models (LLMs) affects the performance of financial information retrieval systems.

The main objective is to determine whether injecting positive (bullish) or negative (bearish) sentiment into financial queries improves retrieval quality or instead introduces semantic drift and measurable ranking bias.

## Research Question

To what extent does sentiment manipulation in LLM-based query expansion alter ranking behaviour and retrieval effectiveness in a hybrid financial information retrieval system?

## Dataset

This project uses the **FiQA test set** from the BEIR benchmark collection, which is automatically downloaded through PyTerrier during execution.

The dataset contains:
- 57,638 financial documents
- 648 queries
- 1,706 binary relevance judgements

The notebook accesses the dataset programmatically, so the raw corpus is not stored in this repository.

## Methodology

The project follows a controlled experimental design composed of four main stages:

### 1. Exploratory Data Analysis

The notebook first analyzes the FiQA corpus and queries to better understand:
- corpus size and document length distribution
- vocabulary characteristics
- financial jargon usage
- lexical mismatch between queries and relevant documents
- relevance sparsity

This analysis motivates the use of semantic retrieval and controlled query expansion.

### 2. Baseline Retrieval

Three standard lexical baselines are tested:
- TF-IDF
- BM25
- BM25 + RM3

These baselines establish a reference point for measuring the effect of more advanced semantic components.

### 3. Hybrid Retrieval Pipeline

The core retrieval architecture is a hybrid pipeline:
- **BM25** retrieves the initial candidate documents
- **Bi-Encoder re-ranking** using `sentence-transformers/all-MiniLM-L6-v2` reorders the candidates based on semantic similarity

This hybrid system serves as the main reference system for all subsequent experiments.

### 4. LLM-Based Query Expansion

Each query is expanded in three controlled modes:
- **Neutral**
- **Positive / Bullish**
- **Negative / Bearish**

The expansions are generated with **Qwen2.5-1.5B-Instruct** and cached in a JSON file for reproducibility.

A separate financial sentiment verification stage uses **FinBERT** to confirm that the generated expansions actually reflect the intended polarity.

## Evaluation Metrics

The following retrieval metrics are used:
- MAP
- Precision@1, Precision@5, Precision@10
- Recall@5, Recall@10
- nDCG@5, nDCG@10

## Main Results

| Method | MAP |
|--------|-----|
| Original | 0.299 |
| Neutral QE | 0.223 |
| Positive QE | 0.185 |
| Negative QE | 0.213 |

## Key Findings

- The original, non-expanded queries achieved the best overall retrieval performance.
- All query expansion strategies introduced a performance penalty.
- Neutral expansion was less harmful than sentiment-driven expansion.
- Positive (bullish) expansion produced the largest drop in performance.
- Negative (bearish) expansion was more robust than positive expansion.
- This suggests that sentiment injection does not simply enrich the query, but can shift its semantic focus away from the original information need.
